# 나만의 종목 분석

**나만의 종목 분석**은 관심 있는 종목이 장기 성장 후보인지 스스로 점검하고, 나만의 종목 판단 리포트를 만들 수 있게 도와주는 교육형 웹앱입니다.

이 프로젝트는 실제 종목을 추천하거나 수익을 예측하는 서비스가 아닙니다.
사용자가 투자 아이디어를 더 신중하게 검토할 수 있도록, 경험 많은 투자자들이 자주 확인하는 기준을 체크리스트와 메모 기반 리포트 형태로 제공합니다.

---

## 프로젝트 목적

많은 개인 투자자들은 유튜브, 블로그, 뉴스, SNS에서 특정 종목을 접한 뒤 감정적으로 투자 결정을 내리기 쉽습니다.

나만의 종목 분석은 이런 상황에서 사용자가 잠시 멈추고 다음 질문들을 스스로 점검하도록 돕습니다.

* 이 회사가 속한 시장은 장기적으로 커질 가능성이 있는가?
* 실제 성장 증거가 있는가?
* 매출 증가가 이익 증가로 이어질 수 있는가?
* 경쟁사가 쉽게 따라오기 어려운 강점이 있는가?
* 현재 가격에 기대감이 너무 많이 반영되어 있지는 않은가?
* 시장이 회사를 다시 평가할 만한 촉매가 있는가?
* 내 투자 아이디어가 틀렸다는 신호는 무엇인가?

## 핵심 기능

현재 버전은 다음 기능을 제공합니다.

* 종목명 또는 티커 기반 검색
* 로컬 종목 목록 기반 선택
* 검색 결과가 없을 때 직접 입력 fallback
* 직접 입력 종목명 40자 제한
* 7가지 투자 판단 기준 기반 체크리스트
* 각 기준별 교육 설명 제공
* 항목별 판단 근거 메모 작성
* 메모 200자 제한
* 작성자의 총평 입력
* 총평 300자 제한
* 100점 만점 종목 판단 점수 계산
* 점수 구간별 해석 제공
* 강점 영역과 추가 확인 필요 영역 표시
* 약점 기반 추가 조사 제안
* Supabase 저장 기반 짧은 공유 링크 생성
* `/r/[id]` 형식의 공유 리포트 페이지
* 기존 `/share?data=...` 공유 링크 호환 유지
* 공유 리포트용 동적 Open Graph 메타데이터
* 종목명, 점수, 점수 라벨이 표시되는 동적 OG 이미지
* 카카오톡, X, Threads 공유 미리보기 지원
* 공유 페이지에서 “내 관심 종목은 몇 점일까?” CTA 제공
* Google Form 기반 피드백 링크
* AI 리포트 초안 기능 관심 버튼
* Vercel Web Analytics 연동
* Supabase 기반 가벼운 클릭 이벤트 로깅
* 투자 추천이 아니라는 고지 표시

## 7가지 체크 기준

이 앱은 다음 7가지 기준으로 종목 아이디어를 점검합니다.

1. **시장 기회**
   회사가 속한 시장이 장기적으로 커질 가능성이 있는지 확인합니다.

2. **성장 증거**
   매출, 수주, 신규 고객, 신제품 등 실제 성장 근거가 있는지 확인합니다.

3. **이익 레버리지**
   매출이 늘어날 때 이익이 더 빠르게 늘어날 수 있는 구조인지 확인합니다.

4. **경쟁 우위**
   기술력, 브랜드, 원가 경쟁력, 고객 락인 등 경쟁사가 쉽게 따라오기 어려운 강점이 있는지 확인합니다.

5. **밸류에이션 부담**
   좋은 회사라도 이미 기대감이 과도하게 반영되어 있지는 않은지 확인합니다.

6. **촉매**
   신제품, 대형 고객, 흑자전환, 산업 사이클 회복 등 시장이 회사를 다시 평가할 계기가 있는지 확인합니다.

7. **리스크와 반증 조건**
   내 투자 아이디어가 틀렸다고 판단할 수 있는 경고 신호를 미리 정리합니다.

## 점수 체계

총점은 100점입니다.

| 기준         |  배점 |
| ---------- | --: |
| 시장 기회      | 15점 |
| 성장 증거      | 20점 |
| 이익 레버리지    | 15점 |
| 경쟁 우위      | 15점 |
| 밸류에이션 부담   | 15점 |
| 촉매         | 10점 |
| 리스크와 반증 조건 | 10점 |

점수 구간은 다음과 같이 해석합니다.

|       점수 | 라벨       |
| -------: | -------- |
| 80 ~ 100 | 강한 검토 후보 |
|  60 ~ 79 | 관심 후보    |
|  40 ~ 59 | 추가 근거 필요 |
|   0 ~ 39 | 신중 검토 필요 |

이 점수는 수익률 예측이 아니라 사용자의 자기 점검 결과입니다.

## 공유 리포트 기능

사용자는 체크리스트와 판단 근거 메모, 작성자의 총평을 작성한 뒤 공유 링크를 만들 수 있습니다.

현재 공유 리포트는 Supabase에 저장되며, 사용자는 짧은 공유 링크를 받습니다.

예시:

`/r/[id]`

공유 리포트에는 다음 정보가 포함됩니다.

* 선택한 종목 정보
* 점수와 라벨
* 강점 영역
* 추가 확인 필요 영역
* 작성자의 총평
* 작성자의 판단 근거 메모
* 7가지 기준별 상세 결과
* 추가 조사 제안
* 투자 추천이 아니라는 고지
* “내 관심 종목은 몇 점일까?” 버튼

현재 기본 공유 방식은 Supabase 저장 기반의 짧은 `/r/[id]` 링크입니다.

기존 URL 기반 공유 방식인 `/share?data=...`도 legacy 링크로 계속 지원합니다.

공유 리포트 페이지는 Open Graph 메타데이터와 동적 OG 이미지를 제공합니다.
이를 통해 카카오톡, X, Threads 등에서 종목명, 점수, 점수 라벨이 포함된 미리보기가 표시되도록 구성되어 있습니다.

## 현재 검증 중인 부분

이 앱은 현재 초기 검증 단계에 있습니다.

현재 핵심 질문은 더 이상 “공유 리포트가 열리는가?”가 아닙니다.
X와 Threads를 통해 공유 리포트 조회는 발생하고 있습니다.

현재 검증해야 할 질문은 다음입니다.

**사용자들이 공유 리포트를 읽기만 하는가, 아니면 읽은 뒤 어떤 행동을 하는가?**

현재 앱은 Supabase `event_logs` 테이블을 통해 다음 클릭 이벤트만 가볍게 기록합니다.

* `share_cta_click`
* `ai_draft_interest_click`
* `feedback_click`

저장하는 데이터는 다음과 같습니다.

* 이벤트 이름
* 페이지 경로
* 공유 리포트 ID가 있는 경우 report id
* 생성 시간

개인 식별 정보는 저장하지 않습니다.

저장하지 않는 정보:

* 이름
* 이메일
* IP 주소
* 로그인 정보
* 브라우저 fingerprint
* 쿠키 기반 사용자 식별 정보

핵심 검증 지표는 다음과 같습니다.

* 공유 리포트 조회수
* `share_cta_click`
* `ai_draft_interest_click`
* `feedback_click`
* 사용자가 새로 생성한 `share_reports` row

공유 리포트 조회수는 계속 증가하지만 클릭 이벤트나 사용자가 새로 만든 리포트가 발생하지 않는다면, 이 프로젝트는 사용자 작성 도구가 아니라 콘텐츠형 종목 리포트 포맷으로 방향을 재검토할 수 있습니다.

## AI 초안 기능 관심 실험

현재 앱은 AI가 자동으로 리포트를 생성하는 기능을 제공하지 않습니다.

대신 향후 기능 가능성을 검증하기 위해 아래 버튼을 제공합니다.

**AI로 리포트 초안 만들기**

이 버튼은 Google Form으로 연결되며, 사용자가 AI 기반 리포트 초안 기능에 관심이 있는지 확인하기 위한 목적입니다.

현재 원칙:

* 아직 AI 생성 기능을 구현하지 않습니다.
* 아직 결제 기능을 추가하지 않습니다.
* 아직 OpenAI 또는 다른 AI API를 연결하지 않습니다.
* 먼저 클릭 이벤트와 폼 응답으로 사용자 관심을 검증합니다.

이 방식은 실제 수요가 확인되기 전에 불필요한 API 비용, 결제 연동 작업, 제품 복잡도가 늘어나는 것을 방지하기 위한 것입니다.

## 투자 관련 고지

이 서비스는 투자 추천이 아닙니다.
제공되는 점수와 해석은 사용자의 자기 점검을 돕기 위한 참고 정보입니다.
모든 투자 판단과 책임은 투자자 본인에게 있습니다.

이 앱은 다음을 제공하지 않습니다.

* 매수 추천
* 매도 추천
* 목표가 제시
* 수익률 예측
* 실시간 주가 분석
* 재무제표 자동 분석
* AI 기반 종목 추천

## 환경 변수

Supabase 저장 기반 공유 리포트와 이벤트 로그 기능을 사용하려면 다음 환경 변수가 필요합니다.

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

주의:

* `NEXT_PUBLIC_SUPABASE_URL`에는 Supabase Project URL만 입력합니다.
* URL 뒤에 `/rest/v1`을 붙이지 않습니다.
* `service_role key`는 사용하지 않습니다.
* `.env.local`은 Git에 커밋하지 않습니다.

## Supabase 테이블

현재 사용 중인 Supabase 테이블은 다음과 같습니다.

### `share_reports`

공유 리포트 데이터를 저장합니다.

| 필드           | 설명                       |
| ------------ | ------------------------ |
| `id`         | 짧은 공유 링크에 사용되는 report id |
| `data`       | 공유 리포트 JSON 데이터          |
| `created_at` | 생성 시간                    |

### `event_logs`

가벼운 클릭 이벤트를 저장합니다.

| 필드           | 설명                  |
| ------------ | ------------------- |
| `id`         | 이벤트 로그 ID           |
| `event_name` | 이벤트 이름              |
| `page_path`  | 이벤트가 발생한 페이지 경로     |
| `report_id`  | 공유 리포트 ID가 있는 경우 저장 |
| `created_at` | 생성 시간               |

현재 기록하는 이벤트:

* `share_cta_click`
* `ai_draft_interest_click`
* `feedback_click`

이벤트 로그는 사용자 행동 전환을 검증하기 위한 최소한의 데이터만 저장합니다.

## 개발 원칙

이 프로젝트는 실제 배포 가능한 MVP를 목표로 합니다.

현재 원칙:

* 기능은 작게 만들되 확장 가능성을 고려합니다.
* Codex는 코드 수정에 집중하고, lint/build/git 명령은 사용자가 직접 실행합니다.
* Git commit과 push는 사용자가 직접 수행합니다.
* 새 기능보다 핵심 사용자 흐름의 안정성을 우선합니다.
* 투자 추천처럼 보이는 문구를 피합니다.
* 공유 리포트는 사용자가 직접 선택하고 작성한 자기 점검 결과로 표시합니다.
* 실제 수요가 확인되기 전에는 AI API, 결제, 로그인, 사용자 계정을 추가하지 않습니다.

## 향후 개선 가능성

현재 버전은 실제 데이터를 자동으로 분석하지 않는 교육형 MVP입니다.

향후 개선 가능성:

* 종목 검색 API 연동
* 더 많은 국내/미국 종목 목록 추가
* ETF 지원
* 결과 저장 히스토리
* 사용자가 직접 입력하는 재무 지표 기반 분석
* 실제 공시/재무 데이터 연동
* AI 리포트 초안 기능
* AI 초안 기능의 유료화 가능성 검증
* 고급 멀티배거 필터
* 글로벌 확장을 위한 영어 UI
* 한국장/미국장 외 시장 확장

이미 구현된 항목:

* 짧은 공유 URL
* Open Graph 메타데이터
* 공유 리포트 미리보기 이미지
* 카카오톡, X, Threads 공유 미리보기
* Vercel Web Analytics
* 클릭 이벤트 로깅

단, 향후 기능이 추가되더라도 서비스의 기본 방향은 투자 추천이 아니라 **투자 판단 보조 및 교육 도구**입니다.
